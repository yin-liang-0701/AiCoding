# 搜索 Tag 功能的架构层次分析

## 1. 需要查看的文件类型

基于典型的 Web 应用架构，搜索 Tag 功能通常会涉及以下层次的文件：

### 表现层（Presentation Layer）
- **前端组件**：搜索框组件、Tag 建议下拉组件
- **API 接口定义**：搜索相关的 API 路由和参数定义

### 应用服务层（Application Layer）
- **搜索服务**：处理搜索请求、调用领域服务
- **Tag 应用服务**：Tag 相关的业务逻辑协调

### 领域层（Domain Layer）
- **Tag 实体/值对象**：Tag 的领域模型
- **Tag 仓库接口**：Tag 数据访问的抽象接口

### 基础设施层（Infrastructure Layer）
- **Tag 仓库实现**：具体的数据库/搜索引擎实现
- **搜索索引服务**：如 Elasticsearch 的索引管理

### 数据层（Data Layer）
- **数据库模型**：Tag 的数据库表结构
- **搜索索引映射**：搜索引擎的索引配置

---

## 2. 各层职责与理由

### 第一层：表现层（前端）
**理由**：用户交互入口，负责输入和展示
- 搜索输入框组件：捕获用户输入
- Tag 建议组件：实时展示匹配的 Tag 列表
- API 调用服务：向后端发送搜索请求

### 第二层：API 网关/路由层
**理由**：请求路由和协议转换
- 接收 HTTP GET/POST 请求
- 参数验证和格式化
- 路由到对应的搜索服务

### 第三层：应用服务层
**理由**：业务流程协调
- 接收并验证搜索参数
- 调用领域服务执行搜索
- 组装返回结果（可能包括权限过滤、结果排序）
- 处理缓存逻辑

### 第四层：领域层
**理由**：核心业务逻辑
- Tag 搜索算法（前缀匹配、模糊匹配等）
- 业务规则验证（如 Tag 状态、可见性）
- 定义搜索 repository 接口

### 第五层：基础设施层
**理由**：具体技术实现
- 数据库查询实现（SQL/NoSQL）
- 搜索引擎实现（Elasticsearch/Meilisearch）
- 缓存实现（Redis）

---

## 3. 修改建议与代码示例

### 推荐架构模式
```
前端组件 → API路由 → 应用服务 → 领域服务 → 基础设施 → 数据库/搜索引擎
```

### 代码结构示例

#### 表现层（API 路由）
```typescript
// routes/search.ts
router.get('/tags/search', async (req, res) => {
  const { query, limit = 10 } = req.query;
  
  // 参数验证
  if (!query || typeof query !== 'string') {
    return res.status(400).json({ error: 'Invalid query' });
  }

  // 调用应用服务
  const results = await tagSearchService.searchTags(query, Number(limit));
  res.json(results);
});
```

#### 应用服务层
```typescript
// services/tagSearchService.ts
class TagSearchService {
  constructor(
    private tagRepository: TagRepository,
    private cacheService: CacheService
  ) {}

  async searchTags(query: string, limit: number): Promise<Tag[]> {
    // 缓存检查
    const cacheKey = `tag_search:${query}:${limit}`;
    const cached = await this.cacheService.get(cacheKey);
    if (cached) return cached;

    // 调用领域层搜索
    const tags = await this.tagRepository.searchByName(query, limit);
    
    // 缓存结果（5分钟）
    await this.cacheService.set(cacheKey, tags, 300);
    
    return tags;
  }
}
```

#### 领域层（接口定义）
```typescript
// domain/tagRepository.ts
interface TagRepository {
  searchByName(query: string, limit: number): Promise<Tag[]>;
  // ... 其他方法
}
```

#### 基础设施层（Elasticsearch 实现）
```typescript
// infrastructure/elasticsearchTagRepository.ts
class ElasticsearchTagRepository implements TagRepository {
  constructor(private esClient: Elasticsearch.Client) {}

  async searchByName(query: string, limit: number): Promise<Tag[]> {
    const { body } = await this.esClient.search({
      index: 'tags',
      body: {
        query: {
          bool: {
            must: [
              { match: { name: { query, fuzziness: 'AUTO' } } },
              { term: { isActive: true } }
            ]
          }
        },
        size: limit
      }
    });
    
    return body.hits.hits.map(hit => this.mapToTag(hit._source));
  }
}
```

## 4. 关键检查点

1. **性能**：是否添加了缓存层？搜索索引是否优化？
2. **安全性**：是否对用户输入进行了 sanitization？
3. **可扩展性**：repository 模式是否便于切换搜索引擎？
4. **错误处理**：各层是否有适当的错误处理？
5. **测试覆盖**：每层是否都有单元/集成测试？

**建议**：如果现有代码将所有逻辑写在单个文件中，建议按上述层次进行重构，使用依赖注入模式解耦各层。
