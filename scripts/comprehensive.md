# repo_reader.py
静态代码分析工具，解析target_repo中所有Python文件（排除测试文件和非必要文件），使用正则表达式提取类和函数定义，建立结构化代码索引并保存到repo_map.json中。

通过build_summary函数为每个文件生成包含路径、函数列表、类列表的摘要信息，形成全局代码结构映射。

# build_index.py
基于TF-IDF的代码索引构建器，将repo_map.json中的代码结构转换为搜索文档语料库。

使用字符级n-gram（而非单词级）进行向量化，更适合代码文本特征，计算每个代码文档的重要性权重。将TF-IDF向量化器、稀疏矩阵和文档映射序列化为.pkl文件，便于快速加载和检索。  

# retrieve.py
**基于TF-IDF索引的代码检索引擎**

加载.pkl文件中的向量化器和稀疏矩阵，使用相同的方式将查询文本转换为TF-IDF向量，计算与所有代码文档的余弦相似度，返回最相关的top-k个代码片段。这是典型的稀疏向量检索RAG系统实现。

# prompt_builder.py
**多策略提示词生成器，为AI助手生成不同详细程度的上下文提示，用于代码仓库分析和任务处理。**

## 工作流程
1. 加载repo_map中的代码索引
2. 通过retrieve模块获取相似度最高的top-k个相关文档（仅包含基本信息）
3. 使用enrich_retrieved_docs将检索结果与repo_map中的完整内容融合（体现RAG系统的经典权衡：检索效率 vs. 数据密度）
4. 根据融合后的上下文生成三种不同策略的提示词

### build_no_context_prompt
零样本学习模式，仅提供任务目标和描述，不附加任何代码上下文，测试LLM的通用理解和推理能力。

### build_rough_context_prompt
提供项目文件结构信息和相关的top-k代码摘要，但不包含完整源码，在防止幻觉和提供上下文之间取得平衡。

### build_structured_context_prompt
提供最完整的上下文：项目背景说明、真实源码片段、函数定义，并通过结构化的方式组织信息，让LLM更容易理解和分析。

# run_experiments.py
**自动化实验运行和评估系统，用于量化评估AI模型在不同上下文条件下的代码理解能力。**

## 核心功能

### 实验执行流程
1. 加载任务定义和代码仓库映射数据
2. 遍历指定任务和模式组合
3. 调用AI API获取回答（支持重试机制）
4. 自动评估回答质量并记录结果
5. 保存回答和评估数据到CSV文件

### 评分系统（核心创新点）
采用**四维连续评分机制**，每个维度0-4分，总分0-10分：

#### 1. 相关文件准确性评分（score_relevant_files函数）
- 基于**精确率、召回率、F1分数**计算，而非简单分档
- 精确率 = 重叠文件数 / 预测文件数  
- 召回率 = 重叠文件数 / 金标准文件数
- F1分数 = 2 × 精确率 × 召回率 / (精确率 + 召回率)
- 额外奖励：完全匹配+0.3分，完全召回+0.2分

#### 2. 结构理解评分（score_structure_understanding函数）
- **explain任务**：侧重关键词覆盖(1.8) + 文件覆盖(1.4) + 流程理解(0.8)
- **feature/bugfix任务**：增加计划性评估(0.6)
- 检测流程词汇："调用"、"流程"、"分层"、"controller"等
- 检测计划词汇："修改"、"新增"、"实现"、"步骤"等

#### 3. 虚构文件惩罚评分（score_nonexistent_files函数）
- **惩罚项**：数量惩罚 + 比例惩罚
- 数量惩罚：虚构文件数 × 1.2（最高3.2分）
- 比例惩罚：虚构文件比例 × 1.2（最高0.8分）

#### 4. 可操作性评分（score_actionable函数）
- **explain任务**：侧重推理和解释能力
- **feature/bugfix任务**：侧重具体实现建议
- 评估维度：回答长度、提及文件数、结构完整性、代码块、推理词汇

#### 总分计算（normalize_total_score函数）
```python
raw = (
    relevant_files_correct +             # 相关文件准确性 (0-4分)
    structure_understanding +            # 结构理解 (0-4分)
    (4.0 - mentions_nonexistent_files) + # 虚构文件惩罚反向计算 (0-4分)
    actionable                           # 可操作性 (0-4分)
)  # 最大值为16分

final_score = raw / 16.0 * 10.0  # 标准化到0-10分
```
